# 인시던트 작성 양식

## 개요
- 문제의 기록, 분석, 대응, 그리고 재발 방지 대책을 체계적으로 작성하기 위한 양식

## 양식
```
타이틀
- 발생한 문제를 간단하고 명확하게 나타내는 제목.

개요
- 인시던트의 전반적인 상황과 주요 이슈를 간략히 설명.

심각도 (Low, Medium, High, Critical)
- 인시던트의 중요도나 심각도를 평가.

증상
- 문제가 발생할 때 나타나는 주요 증상이나 오류 메시지.

영향 범위
- 인시던트가 비즈니스, 서비스, 사용자 등에게 미치는 영향을 설명.

발생 시간
- 인시던트가 발생한 날짜와 시간.

최초 인지 시점
- 문제가 처음 인지된 시점.

대응 시작 시간 
- 문제 대응이 시작된 시간.

현재 상태
- 인시던트의 현재 처리 상태 (예: 대기 중, 조사 중, 대응 중, 완료됨).

담당자
- 문제를 해결하기 위해 대응한 담당자의 이름.

원인
- 문제의 근본 원인을 분석하고 기록.

재현 방법
- 문제가 어떻게 재현되는지 구체적인 방법을 설명.

조치 방법
- 문제를 해결하기 위해 취한 조치와 그 과정.

해결 시간
- 인시던트가 완전히 해결된 시간.

재발 방지 대책
- 인시던트 재발 방지를 위해 필요한 추가적인 조치 또는 개선 사항.

비고
- 추가 참고 사항이나 관련 정보.

참고
- 관련 문서, 로그 파일, 스크린샷 등 문제 분석과 해결에 도움이 되는 자료.
```

## 예시
```
타이틀
- 결제 시스템 오류로 인한 결제 실패 문제

개요
- 고객이 결제 시도를 할 때 결제 시스템에서 오류가 발생하여 결제가 실패하는 문제가 확인됨. 
- 오류 메시지는 "결제 처리 실패"로 나타나며, 여러 사용자에게 영향을 미침.

심각도 (High)
- 결제 기능 장애로 인해 고객의 서비스 이용이 불가능한 상태. 긴급 대응 필요.

증상
- 고객이 결제 시도 시 "결제 처리 실패" 오류 메시지가 화면에 표시됨. 모든 결제 시도가 실패함.

영향 범위
- 전체 고객에게 영향을 미치며, 약 500건의 결제 시도가 실패. 고객 불만 증가와 서비스 신뢰도 저하 우려.

발생 시간
- 2024-09-01, 10:45 AM

최초 인지 시점
- 2024-09-01, 10:50 AM

대응 시작 시간
- 2024-09-01, 10:55 AM

현재 상태
- 완료됨

담당자
- 김철수 (시스템 엔지니어)

원인
- 결제 게이트웨이 서버 업데이트 후 인증서 오류로 인해 결제 요청이 거부됨.

재현 방법
- 결제 페이지에서 카드 결제 시도 시 오류가 발생함. 인증서 확인 절차에서 인증 실패 메시지 확인.

조치 방법
- 결제 게이트웨이 서버의 인증서를 재설치.
- 결제 서버 재부팅 후 서비스 정상화 확인.
- 관련 로그를 분석하여 추가 오류를 점검.

해결 시간
- 2024-09-01, 12:15 PM

재발 방지 대책
- 결제 시스템 업데이트 시 인증서 호환성 검사를 사전 수행하도록 체크리스트 추가.
- 인증서 관련 문제 발생 시 자동 알림 시스템 강화.
- 주기적인 인증서 유효성 점검 및 모니터링.

비고
- 고객 불만을 최소화하기 위해 결제 오류 발생 고객에게 사과 이메일 발송 예정.

참고
- 결제 오류 관련 서버 로그, 인증서 오류 보고서 첨부.
```
